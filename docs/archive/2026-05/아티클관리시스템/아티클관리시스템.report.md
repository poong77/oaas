**이득**: 단일 진실 공급원(DB), 마이그레이션 추적(article_redirects), 검색 인덱싱 통합.

### 4.2 URL 패턴 재정렬: 3-값 URL + 자동 308 리다이렉트

**패턴**: `/help/[product]/[content_type]/[slug]`

**동작**:
- 기존 `/help/[product]/[slug]` 접근 → articles 조회 → content_type 발견 → 새 URL로 308 영구 리다이렉트
- content_type 변경 시 article_redirects 자동 UPSERT → slug가 바뀌었을 때 백업 폴백
- SEO 안정: 301/308 영구, 북마크 수동 갱신 불필요

**이득**: URL 정규화 + 과거 리소스 보존 + 검색 순위 유지.

### 4.3 Tiptap Option A: markdown 저장 + html:true로 스타일 보존

**배경**: commit 48ded32에서 HTML로 저장 전환한 이유는 풀스택 톨바(폰트사이즈/색상/정렬)가 markdown 표준 문법으로 표현 불가능했기 때문. 본 Plan은 마크다운으로 회귀하되 스타일 손실 가드 필수.

**기술**: tiptap-markdown `html: true` 옵션이 이미 활성화되어 있음. `editor.storage.markdown.getMarkdown()` API로 직렬화 시 마크다운 표현 불가능한 노드(폰트사이즈/색상/정렬/밑줄/형광펜/YouTube)를 인라인 HTML 태그로 보존.

**회귀 검증**: `/admin/sandbox/editor-check` 페이지에서 14종 스타일 매트릭스 입력 → 마크다운 저장 → 재로딩 → 스타일 보존 시각 확인.

**이득**: 단일 저장 경로(markdown) → 이메일/SMS 변환 일관성, SES/markdown-to-html.ts 휴리스틱 불필요, 기존 사용처 10곳 전부 통일.

### 4.4 Content Type Enum: 3가지로 고정

```ts
enum articleContentType = ['howto', 'feature', 'troubleshoot']
```

- **howto**: 따라하기. H2 필수: 목표/사전준비/단계/다음단계
- **feature**: 이해하기. H2 필수: 개요/위치/항목설명/관련문서
- **troubleshoot**: 고치기. H2 필수: 증상/원인/해결단계/그래도안되면

**이득**: 메타 검증 + RAG 라우팅 + UI 필터링 명확.

### 4.5 Body Validator: 필수 H2 + 자기완결 본문 검증

```ts
validateBody(markdown: string, contentType: ArticleContentType)
  → { errors: string[], warnings: string[] }
```

- **errors**: H2 누락 → published 차단 (draft는 허용)
- **warnings**: "위에서 말한", "앞서 설명한" 등 → 저장 허용, 표시만

**이득**: RAG 청크 독립 노출 원칙 강제 + 문서 품질 보증.

### 4.6 Keywords 배열 + 3중 검색 매칭

```sql
-- GIN 인덱스 1: keywords 배열
CREATE INDEX articles_keywords_gin ON articles USING gin(keywords);

-- GIN 인덱스 2: tsvector 표현식
CREATE INDEX articles_search_tsv ON articles USING gin(
  to_tsvector('simple',
    coalesce(title,'') || ' ' ||
    coalesce(summary,'') || ' ' ||
    coalesce(body_markdown,'')
  )
);
```

**검색 로직**: q="CI" → synonyms-master 확장 → allTerms 배열 → OR 결합:
1. keywords && allTerms (GIN, 배열 교집합)
2. tsvector @@ to_tsquery (GIN, 풀텍스트)
3. title ILIKE '%q%' (안전망)

**이득**: 동의어/약어/UI 라벨 모두 매칭. "CI" → "체크인" 단어 노출.
