무엇을 바꿨나

기존에는 키워드 5개가 app/_components/home/_constants.ts에 하드코딩되어 있었습니다(홈 Hero + /search 빈 화면 2곳). 이제 어드민이 /admin/master/인기검색어에서 관리하고, 평상시엔 실사용 검색로그에서 자동 집계됩니다.

  • auto — 최근 30일 search_logs.topQueries() 실시간 집계 (DB 저장 안 함, 항상 최신)
  • pin — 어드민이 항상 상단 고정 # 체크인 오류
  • block — 자동집계에서 제외할 노이즈/금칙어 ⊘ test
  • 노출 = pin → auto(block·중복 제외) 병합, 최대 6개 · 1시간 캐시 + 편집 시 즉시 무효화
  • DB 0건 또는 검색기록 0건이면 _constants.ts fallback

마이그레이션 0029_lush_longshot.sql — 신규 enum·테이블·인덱스 2개만 생성. 파괴적 변경 없음. db:generate → db:migrate로 적용 완료 (db:push는 search tsv 인덱스를 드롭하므로 금지).

어드민 화면 (/admin/master/인기검색어)

  • 현재 노출 미리보기 — 사용자가 보는 칩 그대로
  • 고정(pin) 목록 + 정렬/비활성 표시
  • 제외(block) 목록
  • 자동집계 top 15 — 각 검색어에서 한 번에 고정 / 제외 추가 (검색어 prefill)
