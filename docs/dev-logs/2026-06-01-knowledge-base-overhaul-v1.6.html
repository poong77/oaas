💡 인사이트 (다음 사이클로 이월)

정찰의 효과: Phase 5 진입 전 Explore agent로 코드베이스 정찰 결과 image-annotator/ 패키지 + react-konva + 5종 프레임 + KB-10 e2e가 이미 완성되어 있음을 발견. 1주 예상 → 1 commit으로 압축 (80% 단축). DESIGN에 명시된 작업도 실제 코드와 차이가 클 수 있어, 큰 사이클 진입 전 사전 정찰이 결정적.
  • API 일반화는 위임 패턴으로 — anthropic-client에 runClaudeJson 신설 후 callClaudeAssistant를 위임. 외부 시그니처 100% 호환 → 회귀 0.
  • 모드별 rate-limit bucket 분리 — `ai-rewrite-{mode}`. 매니저가 한 모드만 반복해도 다른 모드는 별도 한도 유지. 추후 모니터링도 모드별 가능.
  • 마이그레이션 멱등 패턴이 안전망 — D1·NT-04·0019 모두 IF NOT EXISTS 또는 DO 블록. DB 상태가 schema와 불일치해도 안전 통과.
  • 기존 인프라 검토 = 최선의 중복 회피 — /api/upload 라우트 재사용으로 별도 endpoint 신설 회피. searchArticlesForAutocompleteAction을 D3에서 재사용.
  • prompt cache 임계 4000자 — system 4000자+에서 cache_control 'ephemeral' 활성. SYSTEM_PROMPT 짧게 두면 cache hit 없음 = 효과 0.
  • 섹션별 부분 적용 = markdown-diff의 H2 split이 핵심. body-validator 정규식과 정확 동기 필수 (아니면 섹션 잘못 매칭).
  • ConfirmDialog 중첩 회피 — DiffPreviewModal 자체가 confirm 역할 → 추가 confirm 불필요. 중첩 패턴이 보이면 UX 재설계 신호.
