🎯 오늘의 결과물

호텔리어가 보는 "지금 영업 중인지" 정보를 어드민이 한 화면에서 통째로 관리할 수 있게 만들었다. 영업시간·점심·접수마감·휴무 정책뿐만 아니라 설/추석 단축영업 같은 일시적 변경을 미리 예약해두면 cron이 자동으로 적용·만료시킨다. 변경 이력은 activity_logs에 8종 액션으로 기록된다.

호텔리어 쪽은 헤더 우상단 실시간 배지가 1분마다 라벨을 갱신한다 (영업 중 / 점심시간 / 접수 마감 / 영업 외). /help·/tickets/new에는 우측 sticky ContactPanel이, 사이트 전역 푸터에도 동일한 데이터 소스를 구독하는 가로형 패널이 깔린다. 어드민이 운영시간을 수정하면 호텔리어 쪽이 다음 5분 이내 자동 반영된다 (revalidateTag('business-hours')).

📝 메모

오늘 작업의 가장 큰 가치는 "어드민이 정책을 한 곳에서 통째로 수정"하면 "호텔리어 패널이 자동으로 따라온다"는 단일 진실(SSOT) 구조를 완성한 것이다. 같은 calculateBusinessStatus 순수 함수가 어드민 StatusPreview와 호텔리어 Badge 양쪽에서 호출되므로 preview-truth 1:1 보장. 어떤 화면을 봐도 같은 시점·같은 정책으로 같은 결과가 나온다.

P3에서 양력 공휴일 자동 복제와 슬랙 알림이 들어오면 운영시간 마스터는 완전체가 된다. 그 전까지 어드민이 신경 쓸 일은 (a) 매년 1월에 신년 공휴일 일괄 추가, (b) 명절·점검 일정이 잡히면 미리 예약 등록 — 이 두 가지뿐이다.
