1. 문제 — FAQ 검색이 한 세대 뒤처져 있었다

같은 통합검색 화면이지만 FAQ는 도움말과 다른(약한) 엔진을 쓰고 있었다. 동의어 사전은 물려 있었지만, 사전에 없는 표현이면 그냥 0건이었다.

항목도움말(articles)FAQ (v1.6까지)
키워드/태그 배열keywords text[] + GIN없음
시맨틱(임베딩)embedding vector(1536) 하이브리드없음
인기·유용도 반영—컬럼만 있고 랭킹 미사용
정렬관련도sort_order로 자른 뒤 점수 (버그)

2. 무엇을 했나 (설계 A~D)

항목내용
Akeywords 태그faqs.keywords text[] + GIN. 어드민 에디터에 칩 입력 + AI 추천 버튼(question+answer→한글 키워드). searchFaqs에 arrayOverlaps(keywords) leg.
B인기·유용도 랭킹
+ 정렬 버그 수정		기존 미사용 viewCount(로그, ≤+0.8)·helpfulYes/(yes+no)(최소표본 가드, ≤+0.6)를 점수에 반영. 후보 전체 fetch → score 정렬 → limit으로 버그 수정.
CFAQ 시맨틱 하이브리드faqs.embedding vector(1536) + HNSW. 생성/수정 시 자동 임베딩. searchFaqs 벡터 leg(코사인) 병합, 점수 = 키워드 + cosine_sim × 4 (아티클과 동일). 백필 스크립트.
D0건 질의 → 보강 원클릭검색품질 페이지에 0건 검색어 카드. 각 행에서 "동의어 추가"(canonical 프리필·admin 전용)·"FAQ 작성"(question 프리필)로 바로 연결 → 닫힌 운영 루프.
핵심 설계 원칙: 아티클의 검증된 패턴을 그대로 이식해 두 검색 엔진을 일치시켰다. 인기·유용도 가중치는 관련도(질문일치 +2.5)보다 작게 잡아 관련도가 항상 우선, 동점만 보정. 약어·영문·교차언어는 keywords가 아니라 동의어 마스터가 담당(역할 분리).

3. 동작 방식 — searchFaqs 하이브리드

leg매칭역할
① keywords GINarrayOverlaps(faqs.keywords, 확장term)정확 매칭 가속 경로
② ILIKEquestion · answer 부분일치 (동의어 확장)띄어쓰기·이형어 흡수
③ 벡터embedding <=> query_vec 코사인 최근접사전에 없는 자연어 의미 매칭
세 leg를 id로 병합 → 점수 합산 → 정렬. graceful degrade: OPENAI_API_KEY가 없으면 벡터 leg를 건너뛰고 키워드 검색으로 폴백(서비스 중단 없음, 키 활성 즉시 시맨틱 ON).
