이 와이어프레임에서 검토할 결정사항

MSG-15 — 변수 값 패널의 "소스 선택" UX가 직관적인가?
현재안: 드롭다운(연락처 자동주입 / 직접입력 / 엑셀 열). 대안: 라디오 버튼 3개 나열, 또는 토글 칩. 선택지 명칭이 명확한지 확인 필요.
MSG-16 — 템플릿 편집이 "모달"이 맞는가, 아니면 인라인 슬라이드패널(Sheet)인가?
현재안: 풀스크린 모달. 대안: 우측 Drawer(Sheet)로 열어 목록을 배경으로 볼 수 있게. 본문 에디터가 길어지면 모달 스크롤이 불편할 수 있음.
MSG-17 — 엑셀 컬럼 순서 및 필수/선택 기준이 적절한가?
현재안: 업체명·이메일(필수) + 담당자명·연락처·변수명1~7(선택). 문자용은 이메일 대신 휴대폰 컬럼 필수. 탭별로 서로 다른 템플릿 파일을 제공할지 공용 파일 하나로 할지.
MSG-18 — 호텔리어 전체 불러오기 시 채널 선택 UX는?
현재안: 버튼 클릭 시 간단한 확인 다이얼로그(메일 / 문자 선택). 대안: 탭에 따라 자동 채널 판정(메일 탭이면 이메일, 문자 탭이면 휴대폰). 혼용 불가 업체 처리 방법도 확인 필요.
MSG-19 — 문자 제목 기본값 [오아테크]를 DB 설정으로 뺄 것인가?
현재안: 하드코딩 기본값. 대안: 시스템 설정(어드민 마스터 메뉴)에서 "SMS 기본 제목 접두어" 편집 가능. 브랜드명 변경 시 유지보수 용이성 고려.
MSG-20 — 발신자명이 수신자 메일함에 실제로 표시되는지 SES 설정 확인
AWS SES의 FROM 헤더에 "오아테크" <as@oapms.com> 형태가 실제 동작하는지 샌드박스 테스트 필요. 발신자명에 특수문자 포함 시 인코딩(RFC 2047) 처리 방식도 확인.
변수 값 패널 — 본문에 토큰이 없으면 패널이 사라지는 동작이 맞는가?
현재안: 토큰이 하나라도 있으면 패널 표시, 없으면 숨김. 대안: 항상 표시하되 "변수 없음" 안내. 토큰 삭제 시 패널 행도 실시간으로 사라지는 UX가 직관적인지 검토 필요.
MSG-21 — 문자 미리보기 분할 비율은 좌우 5:5 vs 6:4 중 어느 쪽이 적절한가?
현재안: 5:5 균등 분할 (grid-template-columns: 1fr 1fr). 대안: 6:4(입력 넓게) 또는 4:6(미리보기 넓게). 본문 textarea가 길어질 때 입력 영역이 충분한지, 미리보기 폰 프레임이 너무 작아지지 않는지 검토 필요.
