구현 메모 (확정된 설계)
  • 발송 묶음: 신규 발송부터 batch_id 부여 → 1발송 = 1행. batch_id 없는 기존 이력은 메시지함에 표시하지 않음.
  • 업체명: 발송 시점에 수신자별 업체(호텔)명을 batch에 저장(변수 자동 치환과 동일 매핑 사용). 한 수신자가 여러 호텔이면 호텔명(+N), 직접 입력분은 직접입력.
  • 총발송/성공/실패: batch_id로 group by, status 집계.
  • 유형 구분: 메일 / 문자SMS / 문자LMS / 문자MMS. 문자 세부유형(SMS·LMS·MMS)은 발송 시 본문 byte·제목·이미지첨부로 판정해 batch에 저장.
  • 본문 파라미터: 본문 팝업에 사용된 치환 변수(#{업체명} 등) 칩으로 표시. 본문 아래에 글자수(자/byte)·문자유형 표시.
  • 검색: 발송일(기간), 유형, 업체명, 메일주소, 문자번호. 업체명 검색은 호텔명→연락처→batch 역추적.
  • 페이지당: 20 / 50 / 100. 기본 20.
  • 모바일: 표 대신 카드뷰로 전환(발송일시·유형·총발송·성공/실패·본문보기).

  • 메일 푸터 자동 추가: 발송 시 본문 하단에 회사 푸터((주)오아테크 주소·연락처)를 자동 첨부. 본문 입력 영역에는 미리보기로만 표시(편집 대상 아님), 실제 발송 HTML에 포함. 문자(SMS/LMS/MMS)에는 미적용.
  • 클릭 삽입: 칩 클릭 시 RichEditor(메일)·textarea(문자)의 커서 위치에 #{변수} 토큰 삽입. 커서 없으면 본문 끝에 추가.
  • 기본 변수 세트: 업체명, 담당자명, 연락처, 호텔명 (마스터에서 추가·편집 가능하게 확장 여지).
  • ✅ 발송 시 자동 치환 (확정): 수신자별로 호텔 마스터·사용자 정보의 실제 값으로 치환. 값이 없으면 빈 문자열.

  • 메시지함 업체명도 저장값 사용: 자동 치환을 위해 발송 시점에 업체명을 이미 보관하므로, 메시지함 업체명은 '발송 후 역매핑' 대신 저장된 값을 사용(일관성·정확도↑). 직접 입력분은 직접입력.
  • 메일 vs 문자: 메일은 RichEditor에 삽입, 문자는 byte 수에 영향 → SMS/LMS 자동 전환 표시와 연동.
