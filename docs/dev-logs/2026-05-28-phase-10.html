2. 커스텀 도메인 연결 가이드 (support.oapms.com)

  1. Vercel 대시보드 → poong77/oaas → Settings → Domains
  2. Add → support.oapms.com 입력 → Add
  3. Vercel이 표시하는 DNS 레코드를 OA Tech 도메인 관리자(또는 Cloudflare/Gabia 등)에 등록: 
    Type: CNAME
Name: support
Value: cname.vercel-dns.com.
  4. DNS 전파(보통 5~30분) 후 Vercel이 자동으로 SSL 발급 (Let's Encrypt)
  5. NEXTAUTH_URL 환경변수도 같이 업데이트 필요: 
    npx vercel env rm NEXTAUTH_URL production --yes
echo "https://support.oapms.com" | npx vercel env add NEXTAUTH_URL production
  6. 다음 배포부터 사용자가 https://support.oapms.com로 접속 가능

6. 운영 매뉴얼 — 일상 작업 5가지

① 호텔리어 SSO 가입 후 첫 매핑

OA SSO claim에 hotel_id가 없는 경우 → 가입 보류 상태. 어드민이 /admin/users에서 호텔 매핑 + 권한 설정 → 호텔리어가 다시 로그인하면 정상 접속.

② 장애 발생 시 긴급 배너 노출

  • 방법 A: /admin/service-status에서 상태 incident 선택 → 메시지 입력 → 저장. 즉시 모든 페이지 상단 빨간 배너.
  • 방법 B: /admin/notices에서 kind=incident + 홈 배너 노출 체크 + banner_until 설정. 종료 시각 도래 시 자동 해제.
  • 상태 정상화: /admin/service-status에서 normal로 다시 변경.

③ Dev 에스컬레이션

/admin/tickets/[id] → 사이드바 "Dev 에스컬레이션" → 내용 입력 → 발송. #support_escalation에 별도 스레드 자동 생성 + 티켓에 내부 메모 자동 기록.

④ 콘텐츠 추가 (아티클/FAQ/체크리스트/공지)

매니저+어드민 권한. 모두 어드민 메뉴 → "콘텐츠" 그룹 (아티클/FAQ/체크리스트/공지). 마크다운 split-view 에디터. 발행 토글로 draft/공개 전환.

⑤ 마스터 데이터 변경

어드민만 (단 알림 템플릿/빠른 응대는 매니저+어드민). /admin/master에서 9개 도메인 카드 → 각 CRUD. 변경 즉시 적용.
