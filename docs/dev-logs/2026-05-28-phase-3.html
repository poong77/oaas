2. 신규 DB 스키마

articles

id, product_code, category_path[], slug(UQ), title, summary_30s, body_markdown, toc(jsonb), related_article_ids(uuid[]), author_id, published_at, view_count, helpful_yes/no, +공통(created_at/updated_at/is_active)

  • 인덱스: slug UNIQUE, (product_code, published_at), (is_active, published_at)
  • FK: author_id → users.id ON DELETE SET NULL

article_feedback

id, article_id(FK→articles CASCADE), user_id(FK→users SET NULL), helpful(bool), comment(text), +공통

  • UNIQUE (article_id, user_id) WHERE user_id IS NOT NULL — 로그인 사용자 1회
  • 비로그인 INSERT 무제한 (단순 카운터)

5. 사용자가 실행해야 할 명령 (Phase 3 마무리)

메인 세션 Bash 권한이 어시스턴트에 부여되지 않아 아래 명령은 사용자가 직접 실행해주세요.

# 1. 새 패키지 설치
npm install

# 2. 마이그레이션 생성 (Drizzle이 0001 자동 생성. 이미 손으로 만든 0001 파일은 덮어쓰일 수 있음)
npx drizzle-kit generate

# 3. 마이그레이션 적용 (Neon DB)
npm run db:migrate

# 4. 시드 데이터 보강 (idempotent — 기존 데이터는 유지, 아티클 6건만 추가)
npm run db:seed

# 5. 빌드 검증
npm run build

# 6. 로컬 확인
npm run dev
# http://localhost:3000/help 접속

# 7. 커밋 + 푸시
git add -A
git commit -m "feat: phase 3 — self search (articles + ILIKE search + admin CRUD)

- /admin/articles 어드민 CRUD + 발행 토글
- 도움됨 위젯 + TOC + 30초 요약 + 공유/인쇄

6. 시연 가능한 흐름

  • 1) / 홈 → "최근 업데이트" 섹션에서 발행 아티클 카드 클릭 → 상세 페이지
  • 2) /help → 제품 카드(PMS) 클릭 → /help/pms 리스트 → 아티클 카드 클릭 → 상세
  • 3) GNB 검색창에 "결제" 입력 → /search?q=결제 → 도움말 탭 결과
  • 4) "장애" 탭 클릭 → service_status 검색 (최근 30일, normal 제외)
  • 5) 상세 페이지에서 "도움됐어요" 클릭 → 카운터 증가, toast 표시
  • 6) /admin/articles (manager/admin) → "새 아티클" → split view에서 작성 + 미리보기 → 발행
  • 7) 어드민에서 발행 토글 / 비활성 / 복구 — ConfirmDialog 통한 안전 처리
