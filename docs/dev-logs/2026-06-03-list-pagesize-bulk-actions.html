1. 페이지당 표시 개수 (20 / 50 / 100)

URL 쿼리 ?pageSize= 로 제어. 허용값 화이트리스트(20/50/100)로 임의 주입 차단. 모든 서비스(listUsers/listHotels/listTickets/listNotices/listArticles/listFaqs/listChecklists)는 이미 pageSize(clamp 5–100)를 받고 있어 페이지에서 파싱·전달만 배선.

리스트이전현재셀렉트 위치
사용자 /admin/users고정 2020/50/100하단 페이지네이션 바
호텔 /admin/hotels고정 2020/50/100하단 페이지네이션 바
티켓 /admin/tickets고정 2520/50/100하단(서버 렌더 영역)
공지 /admin/notices고정 2020/50/100하단 페이지네이션 바
아티클 /admin/articles고정 2020/50/100하단 페이지네이션 바
FAQ /admin/faqs고정 2020/50/100하단 페이지네이션 바
체크리스트 /admin/checklists고정 2020/50/100하단 페이지네이션 바

셀렉트는 페이지가 1쪽이어도 항상 노출되도록 푸터 바를 상시 렌더로 변경(기존엔 lastPage>1일 때만 노출). pageSize 변경 시 page=1로 초기화.

2. 사용자·호텔 일괄작업 (소프트 삭제 원칙)

체크박스로 현재 페이지 항목을 선택 → 상단 일괄 작업 바에서 활성화 / 비활성화 일괄 적용. 프로젝트 규칙(물리 DELETE 금지 → is_active=false)에 따라 "삭제"는 비활성화(소프트 삭제)로 구현. 이력·접수기록 보존, 언제든 재활성화 가능.

  • 전체선택 체크박스: indeterminate(부분 선택) 표시 지원, "이 페이지 전체 선택"
  • 데스크탑 테이블 + 모바일 카드뷰 양쪽에 체크박스
  • 실행 전 ConfirmDialog 확인(window.confirm 미사용)
  • 사용자 비활성화 시 본인 계정 자동 제외(자기 잠금 방지) → 토스트에 "(본인 계정 제외)" 안내
  • 감사 로그 activity_logs 기록: user.activate/deactivate, hotel.activate/deactivate (payload.bulk=true, count, ids)

참고 / 주의

  • 선택 상태는 페이지 단위(페이지 이동·필터 변경 시 초기화). 교차 페이지 선택은 미지원.
  • 물리 삭제는 의도적으로 미구현(프로젝트 규칙). 추후 필요 시 FK(티켓·이력) 영향 검토 후 별도 설계.
  • drizzle 배열 연산은 inArray 사용(메모리 규칙 준수).
