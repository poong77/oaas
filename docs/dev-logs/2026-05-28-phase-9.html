4. 홈 동적 전환

app/page.tsx에서 listVisibleQuickActions(), listActiveRoleStarters() 호출 → 빈 결과/오류 시 기존 _constants.ts 하드코딩 fallback. 아이콘은 components/icon-resolver.tsx로 lucide 이름 문자열 → 컴포넌트 매핑.

시연 흐름: 어드민이 /admin/master/quick-actions/new에서 신규 카드 추가 → 홈 새로고침 시 즉시 노출. visible=false로 토글하면 어드민에서는 보이지만 홈에서는 숨김.

5. lib/notifications/templates.ts DB 통합

기존 buildXxx(vars) 빌더 함수는 그대로 유지. 호출부는 결과를 applyDbOverride(eventKey, vars, base)에 통과시키면 DB 템플릿이 있을 때 subject/text/sms를 덮어쓴다. DB row 없으면 base 그대로 반환 (graceful).

6. 시드 보강

  • notification_templates: 10건 (event 5종 × email+sms)
  • quick_actions: 8건 (기존 5개 + 서비스 상태/공지/검색 3종 추가)
  • role_starters: 5건 (front/sales/housekeeping/manager/new_open)
  • system_settings: 5건 (max_upload_mb / rate_limit / slack_channels / business_hours / contact_phone)
  • quick_reply_templates: 3건 (접수 확인 / 재현 요청 / P1 긴급)

모두 idempotent — 재실행 시 0건 신규.

7. 어드민 Nav

admin-nav.tsx에 단일 항목 마스터 데이터 → /admin/master (Database 아이콘) 추가. 매니저+어드민 노출. 기존 8개 메뉴는 그대로 유지.
