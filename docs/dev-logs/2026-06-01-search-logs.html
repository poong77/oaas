1. Executive Summary

관점내용 (실제 산출물 반영)
Problem집계 대시보드(/admin/master/search-quality)는 Hit@3·CTR·0건율을 보여주지만, "어제 누군가 어떤 키워드로 들어와 얼마나 머물고 도움받았는지(또는 이탈/접수했는지)"라는 개별 검색 여정은 볼 수 없었다.
Solution기존 search_logs + articles/faqs.helpful_*를 조회 전용으로 펼쳐 1행=1회로 나열. 사이드바에 인사이트 그룹을 신설하고 검색로그 메뉴를 추가. DB 스키마 변경 0건.
Function UX Effect어제/최근 7일/최근 30일(KST, 오늘 제외) 기간 필터 + 4 StatCard(검색 수·클릭+CTR·티켓 전환·결과없음) + 데스크탑 테이블/모바일 카드뷰/EmptyState + 페이지네이션 + 행별 반응표.
Core Value검색 품질 개선의 정성적 근거 제공 — 0건 이탈 키워드·도움 안 된 콘텐츠를 운영팀이 직접 보고 콘텐츠·동의어·라우팅을 개선.

4. 핵심 구현 포인트

기술 과제해결
기간 경계 (KST)kstPeriodRange — Vercel(UTC)에서도 Asia/Seoul 자정 기준 [start,end). 모든 기간의 end = 오늘 00:00 KST → 오늘 제외.
세션 체류시간같은 session_key 내 LEAD(created_at) 윈도우로 다음 활동까지 간격(초). 다음 없으면 updated_at. 윈도우는 LIMIT 이전 평가 → 페이지 경계 무관.
도움됨 반응표clicked help→articles.slug, faq→faqs.id를 dedupe 후 inArray 배치 조인(페이지당 2쿼리). N+1 없음.
유출 URL 복원buildOutflow — clickedKind별 경로 역산: help /help/{product}/{slug}, faq /faq#faq-{id}, notice /notices/{id}, incident /status, 티켓 /tickets/new, 미클릭 "— (이탈)".
