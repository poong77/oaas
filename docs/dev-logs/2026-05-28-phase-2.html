service_status 운영 흐름

매니저 또는 어드민이 /admin/service-status에서 상태를 변경하면: ① 이전 active row의 is_active=false + ended_at=now() 마감 → ② 새 row INSERT → ③ activity_logs.action='service_status.update' 기록 (fire-and-forget) → ④ /, /status, /admin/service-status revalidatePath → ⑤ 상태가 incident이면 EmergencyBanner가 RSC 단에서 자동 노출.

NT-03 긴급 배너 동작 방식

components/layout/emergency-banner.tsx는 async RSC. 매 페이지 요청 시 getLatestServiceStatus() 호출 → status === 'incident'인 경우만 sticky 빨간 배너 렌더 (z-50). message는 JSX 텍스트 자식으로 렌더되어 React가 자동 escape (XSS-safe). 평상시 normal이면 null 반환 → DOM에 노출되지 않음.
