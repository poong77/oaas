테스트 발송 결과 (토스트)
✅ #support-oa-hotel 으로 테스트 메시지를 보냈습니다.
⚠️ 발송 실패: not_in_channel — 봇이 채널에 없습니다. /invite 후 다시 시도하세요.

📋 동작 정의 (와이어프레임 기준)

  • 채널 검색: 채널명·채널ID 모두 입력 가능. 일반어는 conversations.list 필터, C…/G… 패턴이면 conversations.info로 직접 해석(비공개·미목록 채널도 ID로 연동 가능)
  • 순서: ① 채널명/ID로 찾기 → ② 봇 초대(공개채널 conversations.join) → ③ 첫 메시지 자동 발송 → ④ 로고 회색→컬러
  • 기본정보 변경: 기존 [기본정보 › Slack ID] 단일 필드 제거 → 이 ‘슬랙 채널 연동’ 섹션으로 일원화 (DB hotels.slack_id 컬럼은 폐기·미사용)
  • 연동 성공 판정: botJoined=true (봇이 채널 멤버) — 이 값이 로고 색과 배지를 결정
  • 비공개 채널: 매핑 저장 O, 봇 자동입장 X → ‘봇 미초대’ 표시 + /invite @봇 안내 + [상태 새로고침]
  • 연동해제: soft delete (is_active=false), 이력 보존
  • 중복연동(N:N): 호텔↔여러 채널, 채널↔여러 호텔. unique(hotelId, channelId)로 동일쌍만 차단
  • 접수 알림: 접수 발생 시 botJoined 채널 전체로 buildTicketNewBlocks 발송 (기존 #support_new 병행)
  • 선행 필수: Slack 앱 스코프 channels:read · groups:read · channels:join 추가 + 재설치
