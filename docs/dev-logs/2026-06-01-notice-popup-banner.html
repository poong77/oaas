공지 편집 페이지의 노출 옵션에 "홈 팝업 배너"를 추가했습니다. 켜면 → 이미지 업로드 → 크기(소·중·대) → 종료일자(3일/7일/무기한) → 미리보기. 발행하면 홈 진입 시 이미지 팝업 모달로 노출되고, 클릭 시 공지 상세로 이동합니다. 기존 텍스트 띠 배너(banner)와는 독립이라 동시 사용 가능합니다.

동작 규칙

  • 노출 조건 — 활성 + 발행됨 + 이미지 보유 + popup_until 미경과 (별도 cron 없이 조회 시점 lazy 체크)
  • 이미지 — 기존 ImageUploadDialog + /api/upload(Vercel Blob) 재사용, 새 인프라 없음
  • 크기 프리셋 — 소 max-w-sm · 중 max-w-md · 대 max-w-2xl
  • 3·7일 버튼 — 클릭 시 종료일자를 현재 시각 + N일로 자동 입력
  • 오늘 하루 보지 않기 — 서버 데이터 아닌 브라우저 localStorage(oa-popup-dismiss:{id})
  • 안전장치 — 팝업 켰는데 이미지 없음 / 종료시각 과거 → 저장 전 ConfirmDialog 경고

마이그레이션 정리

⚠️ db:push 금지 함정 확인. npm run db:push는 articles_search_tsv(풀텍스트 GIN 표현식 인덱스, 0015에서 raw SQL 수동 생성)를 DROP하려 한다 — 스키마 미정의로 매번 추적 대상 밖. push의 "Yes"를 누르면 검색 인덱스가 삭제됨.

이번엔 push를 중단하고 popup 컬럼만 멱등 ALTER로 dev DB에 직접 적용했다. 확인 결과 프로덕션용 마이그레이션 0018_gray_blue_marvel.sql이 이전 세션에서 이미 생성·커밋돼 동일한 popup 컬럼을 idempotent하게 담고 있어 배포 시 누락 위험 없음.

결론: 이 프로젝트는 dev에서도 db:push 대신 db:generate + db:migrate 흐름을 사용한다.

수동 확인 권장 (실제 데이터)

  • 공지 편집 → 팝업 켜고 이미지 업로드 → 미리보기로 크기·이미지 확인
  • 3일 / 7일 버튼이 종료일자를 정확히 채우는지
  • 발행 후 홈에서 팝업 모달 노출 + 이미지 클릭 시 공지 상세 이동
  • "오늘 하루 보지 않기" 후 새로고침 시 당일 미노출 / 다음날 재노출
