💡 인사이트 (8건, 다음 사이클로 이월)

  • 사용자 피드백 즉시 반영 패턴 — 큰 Phase 사이사이에 작은 정책 변경을 즉시 commit으로 처리. PLAN/DESIGN v1.0~v1.5로 이력 기록.
  • 모델 ID 미스매치 위험 — 새 모델이 SDK 미등록일 수 있음. 폴백 + ANTHROPIC_MODEL env override가 안전망. 에러에 모델 ID 노출이 진단에 결정적.
  • 마이그레이션 메타 충돌 — drizzle 0015/0016 prevId 같으면 차단. snapshot id 수동 패치로 해결. 한번 직접 SQL 만들면 메타 동기 부담.
  • 표현식 인덱스 부채 — articles_search_tsv처럼 raw SQL로 만든 인덱스는 schema 미정의 → 다음 generate에서 또 DROP. articles.ts에 표현식 인덱스 정의 추가 권장.
  • status enum 보수성 — derived 상태("개선필요")는 메타 컬럼(warning_count) + UI 배지. status 이진성(draft/published) 유지.
  • e2e strict mode — 광범위 role 매치는 strict 위반. level: 1 또는 .first()로 좁히기.
  • AI 1회 호출 + 분산 표시 — 5종 메타를 부분 호출하면 비용 5배. 1회 호출 후 각 필드 옆 분산 표시가 비용·UX 둘 다 최적.
  • 한글/영어 분리 정책 — articles.keywords(한글, 호텔리어 검색어) vs term_synonyms(영어 약어·동의어, 시스템 사전). expandKeywords로 자동 조합.
