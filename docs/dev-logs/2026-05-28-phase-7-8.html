Phase 7 — 공지/업데이트 (NT-01)

1. 데이터 모델 — notices

학습 적용: Phase 5/6의 enum 이름 충돌 경험을 반영해 테이블명(notices)과 enum명(notice_kind)을 명확하게 다르게 명명했습니다.

2. 호텔리어 페이지

  • /notices — placeholder 전면 교체. 종류·제품 필터, pinned 상단 고정, 10건/페이지, 모바일 카드뷰
  • /notices/[id] — MarkdownView 본문, view_count +1, 관련 공지 3건, 인쇄/공유 액션

3. 어드민 페이지 (매니저+어드민)

  • /admin/notices — 검색·종류·제품·발행상태·정렬·페이징·StatCard. /admin/articles 동일 패턴
  • /admin/notices/new + /admin/notices/[id] — NoticeEditor (마크다운 split-view, 핀/배너 옵션, banner_until datetime-local)
  • activity_logs: notice.create / update / publish / unpublish / archive / restore

4. 보강 — 기존 페이지

대상변경
홈 RecentUpdates articles 3건 → pinned notice 1건(별도 카드) + notices 2건 + articles 1건 통합 정렬. 종류 라벨(공지/릴리즈/장애/가이드) 표시
검색 /search "공지" 탭 placeholder 제거. searchNotices() ILIKE 검색 활성화. 카운트 뱃지 / 정렬(관련도·최신·조회수) 지원

Phase 8 — 챗봇 임베드 (CB-01 / CB-02 / CB-03)

1. 컴포넌트 구조

  • components/chatbot/chatbot-fab.tsx — Client Component. 우하단 FAB + 펼침 패널
  • lib/services/chatbot-meta.ts — server-only. OACHAT_EMBED_URL 노출 (env chain 보호)
  • app/layout.tsx — RSC에서 embedUrl 읽고 Client Component에 prop 전달

2. 동작 요약

  • FAB: fixed bottom-3/4 right-3/4 z-40, 둥근 brand 색 버튼, MessageCircle 아이콘
  • 클릭 → 패널 펼침. 데스크탑 w-96 h-[600px], 모바일 풀스크린(inset-0)
  • 모바일 풀스크린일 때 body scroll lock, ESC 키 닫기
  • OACHAT_EMBED_URL 있음 → iframe lazy mount + 로딩 스피너
  • OACHAT_EMBED_URL 비어있음 → fallback 카드 ("챗봇 곧 제공" + 문의 접수 버튼)
  • iframe sandbox: allow-scripts allow-forms allow-same-origin allow-popups

3. 마운트 제외 경로

다음 경로에서는 ChatbotFab 미노출:

  • /login — 인증 화면 방해 방지
  • /admin/* — 운영자 작업 흐름 보호
  • /profile/staff — 직원 관리 UX 보호

4. CB-04 (P2) — from=chatbot 컨텍스트

  • FAB의 fallback 카드 / 하단 안내 모두 /tickets/new?from=chatbot으로 링크
  • TicketCreateForm에서 from=chatbot일 때 본문 상단에 "## 챗봇에서 해결되지 않은 문의" 컨텍스트 prepend
  • customFields.from에 'chatbot' 저장 (추후 Phase 9+에서 chatbot_sessions 연동 시 활용)

시연 흐름

A. 매니저 — 공지 작성 → 발행 → 호텔리어 확인

  1. 매니저 로그인 (manager@oa.local / oa1234!)
  2. 어드민 → 공지 관리 탭 진입
  3. "새 공지" 클릭 → 종류: release 선택 → 제목/본문 작성 → 즉시 발행
  4. 호텔리어로 전환 후 / 홈에서 "최근 업데이트" 위젯 확인
  5. /notices 목록 → 카드 클릭 → 상세 페이지 view_count +1 확인

B. 긴급 배너 — incident 공지

  1. 매니저가 "장애 공지" 작성 시 홈 배너 노출 체크 + banner_until 1시간 후 설정
  2. 모든 페이지 상단에 빨간 띠 노출 → 자세히 보기 클릭 → /notices/[id] 진입
  3. 1시간 경과 후 자동으로 띠 해제 (조회 시점 lazy 체크)

C. 챗봇 FAB

  1. 호텔리어가 홈/help/notices 등 어디든 우하단 챗봇 FAB 확인
  2. 클릭 → 펼침 패널 (모바일 풀스크린, 데스크탑 96x600)
  3. OACHAT_EMBED_URL 비어있음 → fallback 카드 "챗봇 곧 제공" + 문의 접수 버튼
  4. "문의 접수" 클릭 → /tickets/new?from=chatbot → 본문 상단에 "챗봇에서 해결되지 않은 문의" 안내 prepend
  5. /admin/*, /login, /profile/staff에서는 FAB 미노출 확인

D. 검색

  1. 상단 검색창에 "릴리즈" 또는 "결제 지연" 입력 → /search
  2. "공지" 탭 클릭 → ILIKE 검색 결과 표시 (제목 일치 배지)
  3. 결과 없을 때 "문의 접수" 버튼 노출
