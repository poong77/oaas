✅ 요청 사항 이행

  • 아이디 보존 — users.username 신규 컬럼 추가, 원본 ID 그대로 저장 (unique)
  • 이메일·아이디 둘 다 로그인 — credentials 인증을 운영 기본 수단으로 승격, 식별자(이메일/아이디) 통합 입력
  • 전원 호텔리어 + 직책 ‘담당자’ — role=hotelier, title=담당자 일괄 배정
  • 미사용 비활성화 — 사용여부 ‘미사용’ 19건 → is_active=false (소프트, 이력 보존)
  • 기본 비밀번호 123456 — bcrypt cost 12 해시로 저장
  • 첫 로그인 비번변경 안내(강제 아님) — must_change_password=true → 로그인 직후 안내 토스트 + 프로필 배너 (차단 없음)
  • 호텔 연결 — 상세 페이지 user_comp의 compKey를 hotels의 [AS이관 comp_key=NNN] 마커와 매칭 (이름 매칭 불필요, 100% 정확)

⚠️ 짚어야 할 점 — 이메일 부재

“이메일 없으면 호텔 DB에서 가져온다”는 실행 불가했습니다.
hotels 테이블에는 이메일 컬럼 자체가 없습니다(name·phone·managerName·address만 보유). 이관된 업체 마스터(oa-as-companies.json)에도 이메일이 없습니다.

또한 원본 사용자 401명 중 이메일 보유는 단 3명이었습니다(레거시 데이터 실태). 그래서 첫 질문 답변(“아이디로 로그인”) 방침대로 이메일 없는 397명은 {아이디}@as.local 더미 이메일을 넣고, 실제 로그인은 아이디로 하도록 처리했습니다. 더미 이메일로는 메일이 발송되지 않습니다 (이관 시 초대 메일도 발송하지 않음).

👉 추후 사용자별 실제 이메일을 수집하면 프로필/어드민에서 갱신 가능합니다.
