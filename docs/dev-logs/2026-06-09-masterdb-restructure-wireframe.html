4 변경 영향 체크리스트
항목변경위험
인덱스 그리드ITEMS에 group 필드 + 섹션 헤더, 4열, 설명 제거낮음
제품 카테고리product-categories 유지 · 라벨 "제품 분류 → 제품 카테고리"낮음
문의 분류 (신규 통합)ticket-channels + categories → inquiry-classification 부모 + 4탭중간
메시지 템플릿 (신규 통합)notification-templates + quick-replies → message-templates 2탭중간
접수 폼 필드 삭제form-fields 라우트·레지스트리 제거 (참조 여부 확인 필요)확인要
자주 찾는 작업삭제 검토 — 홈/랜딩 사용처·데이터 확인 후 확정확인要
메뉴 구조 → 아티클 메뉴 트리라벨만 변경 (라우트 menu-taxonomies 유지)낮음
홈 큐레이션 → 랜딩페이지섹션 명칭만 변경낮음
구 라우트 redirect물리 삭제 X → 신규로 301 redirect낮음
access map 마이그레이션옛 키 OFF 값 → 신규 키로 OR 병합, 없으면 기본 허용중간
확인 필요 2건: ① 접수 폼 필드가 실제 접수폼/티켓 생성에서 참조되는지(데이터 손실 방지), ② 자주 찾는 작업 삭제 시 홈/랜딩 렌더링 영향. 둘 다 코드·데이터 확인 후 진행.
