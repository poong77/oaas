1 브랜드 컬러 (Green)

어드민·매니저·호텔리어 전 역할 동일. brand-* 토큰은 globals.css가 관리하며 모든 버튼·링크·강조에 쓰입니다.

토큰색상주 용도리뷰 메모
brand-50#f0fdf4아주 옅은 강조 배경·hover 면
brand-100#dcfce7뱃지 배경·선택 칩
brand-300#86efac다크모드 링크·아이콘
brand-500#22c55e다크모드 기본 버튼·포커스 링
brand-600#16a34a라이트 기본 버튼·주요 액션
brand-700#15803d라이트 링크·강조 텍스트·hover
brand-900#14532d다크모드 옅은 강조 배경
brand-950#052e16다크모드 강조 배경 베이스

2 표면 / 배경 (Surface)

카드·패널·섹션·칩의 배경. 라이트 화면 색 → 다크 화면 대응 색.

용도LightDark리뷰 메모
본문 바탕 (body) bg-white dark:bg-slate-950
카드·패널·모달 본문 bg-white dark:bg-slate-900
옅은 섹션·테이블 헤더 bg-slate-50 dark:bg-slate-900
코드칩·뱃지·hover 면 bg-slate-100 dark:bg-slate-800
구분 띠·비활성 트랙 bg-slate-200 dark:bg-slate-800

3 텍스트 (Foreground)

제목→보조설명까지 위계별 글자색. 위로 갈수록 진하고(강조), 아래로 갈수록 흐립니다(muted).

위계 / 용도LightDark리뷰 메모
제목·본문 강조 text-slate-900 dark:text-slate-100
본문 text-slate-800 dark:text-slate-200
본문·라벨 text-slate-700 dark:text-slate-200
보조 본문 text-slate-600 dark:text-slate-300
muted·설명 text-slate-500 dark:text-slate-400
아이콘 흐림·비활성 text-slate-400 dark:text-slate-500

4 테두리 (Border)

카드·인풋·구분선. border-slate-200 → dark:border-slate-700 이 가장 흔한 기본 짝입니다.

용도LightDark리뷰 메모
옅은 구분선 border-slate-100 dark:border-slate-800
기본 카드·인풋 테두리 border-slate-200 dark:border-slate-700
인풋·버튼 테두리(진함) border-slate-300 dark:border-slate-700

5 상태 / 의미 색상 (Status Tint)

옅은 배경 + 진한 글자 → 다크에서 어두운 배경 + 밝은 글자로 반전. 성공·경고·위험·정보·릴리즈에 사용.

의미Light (배경 / 글자)Dark (배경 / 글자)리뷰 메모
성공 · green 50 / 700 950/40 / 300
경고 · amber 50 / 700 950/40 / 300
위험 · red 50 / 700 950/40 / 300
정보 · blue 50 / 700 950/40 / 300
릴리즈 · violet 50 / 700 950/40 / 300

★ 역할별 적용

색상은 세 역할 모두 동일(그린 단일 통일). 다크모드 토글은 모든 역할 공통 제공.

! 변환 제외 항목 (검토 필요)

아래는 의도적으로 다크 짝을 달지 않았습니다. 디자이너 판단이 필요한 부분입니다.

중간톤이라 양쪽에서 읽혀 보존한 것
  • text-slate-300 — EmptyState 화살표·옅은 보조 아이콘 (라이트에서 매우 흐린 장식)
  • text-{red,amber,emerald}-500 — 필수입력 별표·상태 점 (중간 채도, 양쪽 가독 OK)
  • 스위치 손잡이 bg-white — 토글 노브는 다크에서도 흰색 유지
별도 페이즈로 분리한 것
  • landing/ 시안 — Figma 하드코딩 hex 641개. 토큰화 선행 후 다크화 예정(2차)

위 항목들을 다크 대응할지 / 현행 유지할지 메모로 알려주시면 반영하겠습니다.
