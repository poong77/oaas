6. 기존 패턴 준수 체크

  • 물리 DELETE 금지 — 모든 archive/delete는 is_active=false (FAQ·체크리스트·단계).
  • window.confirm/alert 금지 — useConfirmDialog() 사용 (어드민 비활성/복구).
  • DB 호출 페이지에 export const dynamic = 'force-dynamic' 명시.
  • Drizzle pgEnum 이름과 테이블 이름 동명 금지 → checklist_step_action_kind 명명.
  • 모바일 우선 디자인 — 모든 리스트는 데스크탑 테이블 + 모바일 카드뷰 이중 구현.
  • EmptyState 일관성 — 필터 결과 0건 시 "FAQ"/"체크리스트"별 안내 + 문의 접수 CTA.
  • 감사 로그 (활동 로그) fire-and-forget — faq.create/update/archive/restore/move_order, checklist.create/update/archive/restore, checklist.step.create/update/delete/restore/move.

8. 시연 흐름

(A) 호텔리어 — 자가 해결

  • 홈에서 "빠른 해결" 클릭 → /faq 진입
  • 제품 필터 "PMS" 선택 → 2건 표시
  • "PMS 로그인이 안 되는데 어떻게 해야 하나요?" 클릭 → 아코디언 펼침, view_count +1
  • 답변 읽고 "도움됐어요" 클릭 → helpful_yes +1, localStorage에 기록 (재클릭 차단)
  • 답이 부족하면 "트러블슈팅" 버튼 → /troubleshoot
  • "PMS 결제 오류 트러블슈팅" 카드 클릭 → 단계 진행 → "해결됨" 또는 "접수 필요" 결과
  • 접수 필요 시 /tickets/new?from=checklist&checklist={id}&step={no}로 안내

(B) 매니저 — 콘텐츠 관리

  • /admin/faqs → "새 FAQ" → 마크다운 split-view 작성 → 저장
  • 리스트에서 위/아래 화살표로 정렬 변경 (인접 항목과 sort_order swap)
  • /admin/checklists/new → 메타 저장 → /[id]에서 단계 추가 (yes/no 동작 분기 설정)
  • 단계별 위/아래 화살표로 step_no 순서 변경 (unique 제약 우회 위해 임시값 트릭 사용)
