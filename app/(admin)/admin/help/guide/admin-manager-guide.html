통합 AS 플랫폼 — 어드민·매니저 실무 가이드

호텔리어의 문의를 받고, 콘텐츠로 자가해결을 돕고, 시스템을 운영하는 모든 실무를 다룹니다. 신규 사용자가 이 문서 하나로 업무를 시작할 수 있도록 작성했습니다.

대상: 매니저 · 어드민 도메인: support.oapms.com 최종 갱신: 2026-06-01

📖들어가기 전에

이 플랫폼이 무엇이고, 우리(운영팀)가 매일 무슨 일을 하는지부터 잡고 갑니다.

통합 AS 플랫폼이란?

OA 솔루션(PMS·CMS·Keyless·키오스크·웹서비스)을 쓰는 호텔리어를 위한 셀프 서비스 + AS 티켓 허브입니다. 흩어져 있던 3개 서비스를 하나로 통합했습니다.

티켓
기존 as.oapms.com — 문의 접수·처리
아티클·도움말
기존 help.oapms.com (채널.io)
챗봇
oachat.ai — iframe 임베드

운영팀이 하는 일 — 큰 그림

콘텐츠 발행→ 호텔리어 자가해결→ 미해결 시 티켓 접수→ 분류·담당 지정→ 처리·답변→ 상태 알림(SMS/메일)

즉, ① 좋은 콘텐츠를 만들어 문의를 줄이고 → ② 들어온 문의는 빠르고 정확하게 처리하는 것이 핵심입니다.

로그인 & 접근 어드민/매니저 화면은 /admin 경로입니다. OA SSO로 로그인하며(개발환경은 이메일+비밀번호), 권한이 manager 또는 admin인 계정만 진입할 수 있습니다.

👥사용자 3종 & 권한

누가 무엇을 할 수 있는지 먼저 이해하면 화면 구성이 한눈에 들어옵니다. 역할은 hotelier / manager / admin 3종입니다.

호텔리어

이용 고객. 본인 숙소의 티켓 조회·접수, 자가해결 콘텐츠 열람, 본인 숙소 직원 계정 관리.

매니저

OA 운영팀. 전체 티켓 처리, 콘텐츠 편집·발행, SMS/메일 발송, Dev 에스컬레이션.

어드민

OA 관리자. 매니저 권한 전부 + 호텔/계정 관리, 마스터데이터·시스템 설정.

사용자(계정) 관리 /admin/users

매니저·어드민 계정을 만들고, 호텔리어 대표 계정을 발급합니다.

새 계정 만들기

  1. "사용자 추가" 클릭
  2. 이름 · 이메일(중복불가) · 권한(호텔리어/매니저/어드민) 입력 — 호텔리어는 호텔 매핑 필수
  3. 저장하면 임시 비밀번호가 자동 생성되어 이메일+SMS로 발송됩니다
  4. 해당 사용자는 첫 로그인 시 비밀번호 변경이 강제됩니다 (mustChangePassword)
비밀번호 초기화

사용자 상세 → "비밀번호 초기화" → 새 임시비번 자동 발송 + 변경 강제 재설정

비활성화

is_active=false → 로그인 불가, 접수 이력은 보존. 다시 활성화 가능

호텔리어의 직원 관리 (참고)

호텔리어가 직접 /profile/staff에서 본인 숙소 직원을 추가/편집/비활성화합니다. 운영팀이 대신 만들 필요 없이 고객이 셀프로 처리하는 영역입니다. 추가 시 직원에게 임시비번이 자동 발송되고, 자동으로 role=hotelier + 해당 호텔로 묶입니다.

0티켓 관리

먼저 알아야 할 것 — 티켓 상태 4단계

접수 received→ 처리중 in_progress→ 완료 completed ／ 보류 on_hold
  • 접수: 막 들어온 새 문의 · 처리중: 담당자가 진행 중 · 보류: 대기(다시 처리중 복귀 가능) · 완료: 처리 종료
  • 상태를 처리중·완료로 바꾸면 호텔리어에게 SMS/메일 자동 발송됩니다 (연락수단 설정 기준).
  • 모든 상태 변경은 티켓 대화에 "접수 → 처리중" 시스템 메시지로 자동 기록됩니다.

대리 접수 (전화·외부 채널) /admin/tickets/new-by-phone

  1. "대리 접수" 버튼 클릭
  2. 호텔·접수자 선택(선택사항) → 유입 채널 선택(전화 등, 필수)

"직접 접수"(/tickets/new)는 호텔리어 본인이 쓰는 셀프 접수 화면입니다.

2콘텐츠 관리

호텔리어의 자가해결을 돕는 3종 콘텐츠 — 아티클·FAQ·체크리스트. 좋은 콘텐츠는 곧 줄어든 티켓입니다.

⭐ 핵심 개념 먼저 — 안 읽으면 헷갈립니다

① 아티클(Article)이란?

호텔리어가 보는 제품 사용 핸드북 콘텐츠입니다. 제품별·메뉴 경로별로 분류되며, 마크다운으로 작성합니다. (기존 help.oapms.com을 대체)

고객 측 노출: /help/[제품]/[의도타입]/[slug] — 목차·요약·도움됨 피드백·관련문서 포함.

② ⭐ 의도 타입(Intent Type) — 가장 많이 묻는 개념

아티클을 만들 때 "독자가 이 글을 왜 읽는가"를 3가지로 분류합니다. 선택하면 본문 뼈대(필수 H2)가 자동으로 깔립니다.

아티클 작성·발행 /admin/articles

  1. 제품 선택
  2. 의도 타입 선택 → 본문 뼈대 자동 주입
  3. 메뉴 경로 선택 (제품별 카테고리 트리 — 미선택 시 /help 사이드바에 안 뜸)
  4. 제목(60자↓) · Slug(영문/숫자/하이픈, 전역 유일 — "운영 ID 채번"·"검증" 버튼 제공)
  5. 요약(200자 권장) · 키워드 3개↑ (검색·동의어 확장에 쓰임)
  6. 본문 작성 (마크다운, 자동저장) → 발행 (발행 즉시 고객 노출)
발행 차단(Hard)

제품·의도타입·제목·Slug·본문 누락 시 발행 불가

필수 H2 누락·키워드 부족·메뉴경로 미선택 → "보완 N건" 배지(발행은 가능)

  • 발행 상태: draft ↔ published. 발행 후 수정도 즉시 반영.
  • 도움됨 👍👎, 조회수, 관련 아티클, 자동 목차(TOC) 지원.
  • 키워드는 검색 동의어 사전과 연동됩니다(마스터 §동의어).

FAQ 작성 /admin/faqs

자주 묻는 질문을 짧은 Q&A로. Draft 개념 없이 저장 즉시 노출됩니다.

  • 입력: 제품(필수) · 문제 유형(선택) · 질문 · 답변(마크다운) · 정렬 순서
  • 고객 측: /faq 아코디언 UI (제품·유형 필터, 도움됨 피드백)
  • 삭제는 비활성화로 (물리 삭제 금지)

호텔리어가 예/아니오로 단계를 따라가며 스스로 진단·해결하는 도구입니다. 각 단계의 응답에 따라 다음 행동이 갈립니다.

4OA 챗봇 관리 (외부)

oachat.ai 챗봇을 iframe으로 임베드합니다. 챗봇 자체 콘텐츠는 외부 서비스에서 관리하고, 우리 쪽은 연결·노출만 담당합니다.

  • 호텔리어 화면 우하단 플로팅 버튼(FAB)으로 노출 (모바일은 풀스크린).
  • 연결 키: 환경변수 OACHAT_EMBED_URL. 이 값으로 임베드 주소를 설정합니다.
  • 로그인·어드민·직원관리 화면에서는 노출 안 됨 (호텔리어 모드에서만).
장애 대비 Fallback임베드 주소가 비어있거나 챗봇 장애 시, "지금은 문의 접수를 이용해주세요" 안내와 함께 /tickets/new?from=chatbot 접수 링크를 보여줍니다. 챗봇이 죽어도 고객은 접수할 수 있습니다.

※ 챗봇 답변 내용/지식 베이스는 oachat.ai 콘솔에서 관리합니다(본 플랫폼 외부).

5연동 — Slack · SMS · 메일

티켓 이벤트가 외부로 자동 전파되는 구조입니다. 대부분 자동이라 설정만 알면 됩니다.

Slack 연동

티켓 이벤트가 운영 Slack 채널로 자동 발송됩니다(Webhook 방식). 메시지엔 "큐에서 열기" 버튼이 붙습니다.

이벤트발송 채널환경변수

Webhook URL은 .env에만 저장(절대 커밋 금지). 미설정 시 실제 발송 없이 로그만 남고 메인 로직엔 영향 없습니다.

SMS(솔라피) & 이메일(AWS SES)

호텔리어가 접수 시 고른 연락 수단(SMS/메일)에 따라 자동 발송됩니다.
